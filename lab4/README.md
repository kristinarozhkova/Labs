Після збереження розробленого скрипту було запущено створений сервер та виконано кілька запусків клієнтського скрипту для перевірки роботи (рис. 3).

3. Передача файлів між клієнтом і сервером

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Останім етапом лабораторної роботи стало розробка TCP-серверу, який буде здатний приймати текстовий файл від клієнта та зберігати його. На відміну від echo-сервера, цей сервер працює не з текстовими повідомленнями, а з довільними даними у форматі байтів. Після встановлення з’єднання сервер переходить у режим приймання даних та записує отриманий вміст у файл на диску.
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Для тестування роботи серверу було створено клієнтський скрипт, який виконаує надсилання інформації, яка міститься в зазначеному файлі на серевр через TCP-з'єднання. На рисунку 4 представлено запуск клієнтського скрипту.
