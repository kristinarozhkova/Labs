Лабораторна робота №5: Unit-тестування програм
--------------------------------------------------------------------------- 
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; У межах даної лабораторної роботи було виконано налаштування середовища для unit-тестування програмного забезпечення мовою Python з використанням бібліотек pytest та pytest-cov. Для цього попередньо було встановлено необхідні залежності за допомогою команди "pip install pytest pytest-cov".

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Після успішного встановлення необхідних інструментів було реалізовано об’єктно-орієнтовану програму, що моделює роботу бібліотеки. Програма складається з класів Book та Library і містить методи для видачі та повернення книг, додавання книг до бібліотеки, а також отримання списку доступних книг. Скрпит реалізованого програмного забезпечення  представлені в файлі library.py.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Далі було виконано створення файлу unit-тестів з назвою test_library.py, у якому реалізовано unit-тести для кожного методу програми. Для кожного методу було написано щонайменше два тести, які перевіряють коректну роботу методів та обробку помилкових ситуацій. Після цього було виконано запуск unit-тестів за допомогою команди pytest. На рисунку 1 представлено результат виконання unit-тестів у командному рядку.

Рисунок 1 – Результат виконання unit-тестів

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Отриманий результат, наведений на рисунку 1, свідчить про успішне виконання всіх unit-тестів без помилок, що підтверджує коректність реалізації програмного коду та тестів. Далі було виконано перевірку покриття програмного коду unit-тестами з використанням бібліотеки pytest-cov. Для цього було використано команду pytest --cov=library. У результаті виконання даної команди було отримано звіт у консолі, який відображає рівень покриття програмного коду тестами. На рисунку 2 наведено результат перевірки покриття.

Рисунок 2 – Результат перевірки покриття програмного коду unit-тестами

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Згідно з отриманими даними, рівень покриття програмного коду unit-тестами становить 100%, що перевищує мінімальну вимогу завдання у 85%. Це свідчить про те, що всі основні гілки виконання програми були перевірені під час тестування.

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Завершальним етапом лабораторної роботи стало формування HTML-звіту покриття програмного коду unit-тестами. Для цього було використано команду pytest --cov=library --cov-report html. У результаті було згенеровано HTML-звіт у папці htmlcov, який наочно відображає виконання кожного рядка програмного коду. &nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Таким чином, у ході виконання лабораторної роботи було реалізовано об’єктно-орієнтовану програму, написано unit-тести для кожного методу, перевірено покриття програмного коду тестами та сформовано HTML-звіт. Усі вимоги завдання було виконано у повному обсязі.
